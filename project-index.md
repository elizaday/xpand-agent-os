# Project Index

Use this file to track where project-specific context lives.

## Master Context Files

- `AGENTS.md`: working instructions for Codex when this folder is opened as a project
- `profile.md`: durable personal and professional context
- `voice.md`: writing style rules
- `strategy.md`: Xpand Studio and larger decision-making context
- `active-projects.md`: current projects and status notes

## Project-Specific Context To Create

Each active code or writing project should eventually get its own short context file. Do not copy the entire personal context into every repo. Extract only what matters.

Recommended template:

```md
# Project Name

Status:
Last updated:
Owner:
Collaborators:
Workspace/repo:
Stack:

## Goal

What this project is supposed to do.

## Current Next Step

The one thing that matters next.

## Definition Of Done

- Clear outcome
- Build/check requirement
- Handoff or documentation requirement

## Notes For Codex

Project-specific instructions, constraints, and voice/product rules.
```
