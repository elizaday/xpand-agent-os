# Memory Policy

This project uses both tool memory and human-readable memory.

## Working Memory

Use `memory/working-memory.md` for short-term context that matters this week or this project cycle.

## Durable Memory

Use `memory/durable-memory.md` for stable facts about Elizabeth's work, preferences, identity, and recurring patterns.

## Decisions

Use `memory/decisions.md` for decisions that should guide future work.

## Stale Or Unverified

Use `memory/stale-or-unverified.md` for claims that need checking before use.

## Rules

- Do not treat old dates as current.
- Do not silently overwrite context when new information conflicts with old information.
- Record important decisions in plain language.
- Keep memory useful, not exhaustive.
- Prefer verified, current context over remembered context.
