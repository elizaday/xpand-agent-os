# Project Triage Skill

## Trigger

Use this when Elizabeth asks for project status, next steps, or a read on where things stand.

## Inputs

- Project files or notes.
- Current date.
- Known deadlines.
- Relevant repo or workspace.

## Process

1. Verify current files, docs, and status before summarizing.
2. Separate stable work from blocked, stale, or speculative work.
3. Identify the one next action that would create the most clarity.
4. Flag date-sensitive claims that need verification.

## Output

- Stable
- Blocked or stale
- Next action
- Verification needed
